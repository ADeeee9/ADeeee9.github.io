1.4、CSS的3种导入方式

1.行内样式 2.内部样式 3.外部样式

拓展 : 外部样式两种写法

<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <title>Title</title>

    <!-- 内部样式 -->
    <style>
        /*CSS的注释*/
        h1{
            color: green;
        }
    </style>

    <!-- 外部样式:链接式 -->
    <link rel="stylesheet" href="css/style1.css">

</head>
<body>

<!-- 行内样式: 在标签元素中，编写一个style属性，编写样式即可
     优先级: 行内样式 > 内部样式 > 外部样式
     优先级: 就近原则--谁离这个元素近，就使用谁   -->
<h1 style="color: red">我是标题</h1>

</body>
</html>

  • 链接式：

    HTML

    <!-- 外部样式 -->
<link rel="stylesheet" href="css/style1.css">

  • 导入式：

    @import是CSS2.1特有的！

    <!-- 导入式 -->
<style>
    @import "css/style1.css";
</style>

    导入式不常用，且导入式为先显示网页结构，再渲染。

  • 补充：

    首页link和import语法结构不同，前者是HTML标签，只能放入HTML源代码中使用，后者可看作为CSS样式，作用是引入CSS样式功能。import在HTML使用时候需要标签，同时可以直接"@import url(CSS文件路径地址);“放如CSS代码里引入其他CSS文件。

    本质上两者使用选择区别不大，但为了软件中编辑布局网页HTML代码，一般使用link较多，也推荐使用link。

2、选择器

作用：选择页面上的某一个或者某一类元素

2.1、基本选择器

  • 1、标签选择器

会选择到页面上所有的这类标签的元素

标签名{}

  • 2、类选择器 - class

选择所有class属性一致的标签

好处：使用class属性分类，可以跨标签进行修饰，也可以多个同类标签不同修饰

.class名称{}

  • 3、id选择器

id全局唯一

#id名称{}

  • 4、补充

优先级 ：id选择器 > 类选择器 > 标签选择器

2.2、层次选择器

  • 1、后代选择器：指向某个元素的后面的所有后代元素

  • /* 后代选择器*/
body ul li p{
    background: red;
}

  • 2、子选择器：指向某个元素的一个子元素 (爷爷>父亲)（父亲>儿子）

  • /* 子选择器 */
body>p{
    color: #c8ee27;
}

  • 3、相邻选择器：同辈，相邻（对下不对上），只有一个兄弟元素

  • /* 相邻选择器 */
.active + p{
    background: blueviolet;
}
    <p class="active">标题1</p>
    <p>标题2</p>

  • 4、通用选择器：当前选中元素向下的所有兄弟元素，不止一个，不包含自身

  • /* 通用选择器 */
.active2 ~ p{
    background: red;
}
    <p class="active2">标题8</p>
    <p>标题9</p>
    <p>标题10</p>

    2.3、结构伪类选择器

    /* ul的第一个子元素 */
ul li:first-child{ /*伪类*/
    background: #c8ee27;
}
/* ul的最后一个子元素 */
ul li:last-child{/* 选择副级的第一个元素 */
    color: red;
    background: blueviolet;
}

/* 选中p1: 定位到父元素，选择当前的第一个元素
   选择当前p元素的父级元素，选中父级元素的第一个，并且是当前元素才生效！ */
p:nth-child(1){
    background: #1fbebe;
}

p:nth-of-type(2){/* 选择父元素中第二个类型为p的元素 */
    color: blue;
}

a:hover{/* a标签被选中后的表现 */
    background: #c8ee27;
}

5.4、父级边框

  • 如何解决父级边框塌陷问题

1、增加父级元素的高度

#father{
 border: 1px #000 solid;
}

2、增加一个空的div标签，清除浮动

<div class="clear"></div>

.clear{
 clear: both;
 margin: 0;
 padding: 0;
}

3、overflow

​ 在父级元素中增加一个 overflow：hidden;

4、父类添加一个伪类：after

#father:after{
 /*用content增加一个空内容*/
 content: '';
 display: block;
 clear: both;
}

  • 小结：

    1、设置父元素的高度

    ​ 简单，但元素假设有了固定的高度，就会被限制

    2、浮动元素后面增加空div

    ​ 简单，但代码中尽量避免空div

    3、overflow

    ​ 简单，但在下拉的一些场景或有内容不可隐藏时避免使用

    4、父类添加一个伪类：after（推荐）

    ​ 写法稍微复杂，但是没有副作用，推荐使用！！！

6、定位

6.1、相对定位

<!--相对定位
 相对于自己原来的位置进行偏移
 -->
<style>
  div{
    magin: 10px;
    padding: 5px;
    font-size: 12px;
    line-height: 25px;
  }
  #father{
    border: 1px solid #666;
    padding: 0;
  }
  #st{
    border: 1px solid #c8ee27;
    background-color: blueviolet;
    position: relative;/*相对定位: 上下左右*/
    top: -20px;/*距离原位向下减20，相当于向上移20像素*/
    left: 20px;/*距离原位向左加20，相当于右移20像素*/
  }
  #nd{
    border: 1px dashed #1fbebe;
    background-color: #c8ee27;
  }
  #rd{
    border: 1px dashed #ff008a;
    background-color: #41d04d;
  }
</style>

相对定位：position：relative；

相对于原来的位置，进行指定的偏移。

相对定位后，原来的位置仍然被保留，仍然在标准文档流中。

6.2、绝对定位

#nd{
    border: 1px dashed #1fbebe;
    background-color: #c8ee27;
    position: absolute;
    right: 50px;
}

两种情况

1、父级元素没有定位的前提下，会相对于浏览器定位

2、假设父级元素存在定位，我们通常会相对于父级元素进行偏移，会在父级元素范围内移动

定位：基于xxx定位，同样为上下左右4个方向

相对于父级或浏览器的位置，进行指定的偏移

绝对定位后不在标准文档流中，原来的位置不会被保留

6.3、固定定位

<style>
    body{
        height: 1000px;
    }
    div:nth-of-type(1){
        /* 绝对定位
        没有父级元素，相对于浏览器定位
        */
        width: 100px;
        height: 100px;
        background: #41d04d;
        position: absolute;
        right: 0;
        bottom: 0;
    }
    div:nth-of-type(2){
        /*fixed，固定定位
        相对于初始位置定位
        随页面滚轮滑动不会改变位置
        */
        width: 50px;
        height: 50px;
        background: #c8ee27;
        position: fixed;
        right: 0;
        bottom: 0;
    }
</style>
